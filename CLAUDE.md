## Project Overview

NeoLabs AI is a Laravel 13 + Inertia v2 + React 19 SPA. Authentication is handled by Laravel Fortify (with 2FA support). The frontend uses shadcn/ui (New York style) with Radix primitives, Tailwind CSS v4, and Lucide icons. Vite 8 bundles the frontend with React Compiler enabled via Babel plugin. Laravel Wayfinder generates TypeScript route bindings from PHP routes. Served locally via Laravel Herd at `https://neolabs-ai.test`.

## Architecture

### Backend (PHP)

- **Controllers**: `app/Http/Controllers/` — organized by domain (e.g., `Settings/ProfileController.php`)
- **Form Requests**: `app/Http/Requests/` — mirrored by domain, always used instead of inline validation
- **Actions**: `app/Actions/Fortify/` — Fortify action classes (CreateNewUser, ResetUserPassword)
- **Concerns**: `app/Concerns/` — shared traits (PasswordValidationRules, ProfileValidationRules)
- **Models**: `app/Models/` — Eloquent models
- **Providers**: `app/Providers/` — AppServiceProvider and FortifyServiceProvider
- **Routes**: `routes/web.php` (main), `routes/settings.php` (settings pages), `routes/video-editor.php` (video editor), `routes/console.php`
- **Services**: `app/Services/` — domain services (e.g., `VideoEditor/FFmpegService.php`, `VideoEditor/CaptionService.php`)
- **DTOs**: `app/DTOs/` — data transfer objects by domain (e.g., `VideoEditor/ApplyToolData.php`)
- **Jobs**: `app/Jobs/` — queued jobs (e.g., `ProcessVideoEditJob.php`)
- **Enums**: `app/Enums/` — PHP enums (e.g., `VideoEditorTool.php`, `VideoEditSourceType.php`)

### Frontend (TypeScript/React)

- **Pages**: `resources/js/pages/` — Inertia page components, directory structure maps to routes (auth/, settings/, dashboard.tsx, welcome.tsx)
- **Layouts**: `resources/js/layouts/` — app-layout.tsx (authenticated), auth-layout.tsx (guest), with sub-layouts in app/ and auth/ dirs, plus settings/
- **Components**: `resources/js/components/` — app-specific components
- **UI Components**: `resources/js/components/ui/` — shadcn/ui primitives (do not manually edit, use `npx shadcn@latest add`)
- **Hooks**: `resources/js/hooks/` — custom React hooks (use-appearance, use-clipboard, use-two-factor-auth, etc.)
- **Types**: `resources/js/types/` — shared TypeScript types, re-exported from index.ts (auth.ts, navigation.ts, ui.ts)
- **Wayfinder**: `resources/js/actions/` and `resources/js/routes/` — auto-generated by Wayfinder plugin, never edit manually
- **Path alias**: `@/*` maps to `resources/js/*`

### Testing

- **Framework**: Pest v4 with `RefreshDatabase` applied to all Feature tests (configured in `tests/Pest.php`)
- **Structure**: `tests/Feature/` (most tests), `tests/Unit/` — mirrors app domain structure (Auth/, Settings/)
- Create tests with: `php artisan make:test --pest {name}`

### Key Conventions

- TypeScript imports must use `type` imports for type-only values (enforced by eslint `consistent-type-imports`)
- Import order is enforced: builtin → external → internal → parent → sibling → index (alphabetized)
- Blank lines required around control statements in TS (enforced by `@stylistic/padding-line-between-statements`)
- Curly braces always required for control structures in both PHP and TS
- PHP formatting uses Laravel preset (`pint.json`)
- React Compiler is enabled — follow React rules of hooks strictly
- Wayfinder is configured with `formVariants: true` — generates form action helpers alongside standard route bindings
- SSR entry point exists at `resources/js/ssr.tsx`; use `composer run dev:ssr` for SSR development

## AI Module Architecture

New AI feature modules (chat, image generation, presentations, documents, transcription) should follow the pattern established by the Video Editor module:

- **Controller** (thin) → **Form Request** (validation) → **Service** (orchestration) → **Actions** (single-purpose logic)
- Use **DTOs** for structured payloads between layers
- Use **Jobs** for async processing (image gen, transcription, document export, video rendering)
- Track job status: pending → processing → completed → failed
- All AI provider calls go through dedicated Service classes — never call providers directly from controllers
- Abstract provider logic to allow switching providers; normalize responses into internal DTOs
- Store generated assets in cloud storage (S3); save metadata (path, status, type, user_id) in database; use signed URLs for access
- Keep modules isolated — do not mix logic between modules
